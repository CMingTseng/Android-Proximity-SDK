ProximityObserver

interface ProximityObserver

The basic object for proximity observation. It lets you define the zones in which your actions will be called, when defined preconditions were met.

Each zone is strictly connected to an attachment key/value pair. Many beacons can have the same attachment pairs associated, making your zone consist more than a one beacon.

You can define as many zones as you need, using ProximityObserver.zoneBuilder for that. Use ProximityObserver.addProximityZone or ProximityObserver.addProximityZones to add created zones to the ProximityObserver. There will be no interaction with bluetooth scan until you will call ProximityObserver.start method. After that, the scan will be started, and you will receive a Handler object that you should use to stop the scanning later on. It is suggested to keep the reference to the Handler for as long as the scan should persist.
