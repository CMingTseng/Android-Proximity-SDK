withOnChangeAction

abstract fun withOnChangeAction(onChange: (List<ProximityAttachment>) -> Unit): ProximityZoneBuilder

This action will be called whenever a change occurs inside a zone. Let's say we have two beacons - A and B, both having the same zone defined by key/value pair "venue":"office". If somebody enters the proximity of beacon A, the onEnterAction is called for the whole zone. If that person walks away from beacon A and goes to beacon B, the zone is still maintained, so no onExitAction will be called yet. However, there was a change inside the zone, so the current nearby attachments have changed (from beacon A to B). This is where onChangeAction gets called. It helps you to track the current nearby attachments inside a zone.

Parameters

onChange - the lambda/func1 action to invoke when a change occurs. Takes a list of the current attachments nearby as a parameter.
